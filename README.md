Созданы группы пользователей
1. sudo groupadd название
Эта команда создает новую группу с именем, указанным вместо "название". Например, если вы хотите 
создать группу с именем "developers", команда будет выглядеть так: sudo groupadd developers.

2. sudo adduser testuser
Эта команда создает нового пользователя с именем "testuser". В процессе выполнения команды вам 
может быть предложено задать пароль и некоторые другие параметры (например, полное имя).

3. sudo usermod -aG sudo(группа) testuser
Эта команда добавляет пользователя "testuser" в группу "sudo". Флаг -a (append) означает, что пользователь 
будет добавлен в группу без удаления его из других групп. Это позволит "testuser" выполнять команды с 
правами суперпользователя, используя команду sudo.

4. getent group developers
Если группа существует, вы получите строку с информацией о ней, которая будет выглядеть примерно так:

5. groups имя_пользователя
Проверка групп пользователя

6. sudo gpasswd -d имя_пользователя имя_группы 
Удаление пользователя из группы

# 1. Настройка параметров ОС, драйверов и служб

### 1.1 Использование специальных утилит для настройки ядра (2,5 балла)

- Утилита: `sysctl` — для настройки параметров ядра

```bash
# Просмотр текущих параметров ядра
sudo sysctl -a | less

# Пример настройки параметра (увеличение максимального количества открытых файлов)
sudo sysctl -w fs.file-max=100000

# Для постоянного применения параметра
echo "fs.file-max=100000" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

- Утилита: `modprobe` — управление модулями ядра

```bash
# Просмотр загруженных модулей
lsmod

# Загрузка модуля (пример: модуль для ext4)
sudo modprobe ext4

# Добавить модуль в автозагрузку
echo "ext4" | sudo tee -a /etc/modules
```

---

### 1.2 Настройка сетевого протокола SSH (2 балла)

- Установка SSH-сервера

```bash
sudo apt update
sudo apt install -y openssh-server
```

- Проверка статуса SSH

```bash
sudo systemctl status ssh
```

- Настройка конфигурации SSH (пример: запрет входа root)

```bash
sudo nano /etc/ssh/sshd_config
# В файле найти и изменить:
PermitRootLogin no
# Сохранить и выйти

sudo systemctl restart ssh
```

- Проверка, что SSH слушает на порту 22

```bash
sudo ss -tlnp | grep sshd
```

### 1.3 Настройка удаленного доступа к активной сессии (2 балла)

- Для доступа к активной сессии через SSH можно использовать `tmux` или `screen`

```bash
sudo apt install -y tmux
```

- Запуск сессии tmux

```bash
tmux new -s mysession
```

- Подключение к сессии с другого терминала

```bash
tmux attach -t mysession
```

- Альтернативно, можно использовать VNC для графического доступа:

```bash
sudo apt install -y tigervnc-standalone-server
```

Настроить VNC-сервер и подключаться к активной сессии.

# 6. Реализация защиты компьютерных систем

### 6.1 Резервное копирование установленной операционной системы (2 балла)

- Создадим образ системы с помощью `rsync` (пример):

```bash
sudo rsync -aAXv --exclude={"/dev/*","/proc/*","/sys/*","/tmp/*","/run/*","/mnt/*","/media/*","/lost+found"} / /backup/ubuntu_backup/
```

- Или с помощью `tar`:

```bash
sudo tar --exclude=/backup --exclude=/proc --exclude=/sys --exclude=/dev --exclude=/tmp -cvpzf /backup/ubuntu_backup.tar.gz /
```

(путь /backup должен существовать и иметь достаточно места)

---

### 6.2 Создание установочного образа системы (1 балл)

- Утилита `dd` для создания образа диска (пример):

```bash
sudo dd if=/dev/sda of=/backup/ubuntu_disk_image.img bs=4M status=progress
```

- Для создания ISO образа установленной системы можно использовать инструменты типа `remastersys` или `Systemback` (устаревшие) или `pinguybuilder`.

---

### 6.3 Создание точек восстановления системы (2 балла)

Ubuntu не имеет встроенных точек восстановления, но можно использовать `Timeshift`:

```bash
sudo apt install -y timeshift
sudo timeshift --create --comments "Initial snapshot" --tags D
```

- Для создания и управления снимками:

```bash
sudo timeshift --list
sudo timeshift --restore
```

---

### 6.4 Создание групп пользователей (1 балл)

```bash
sudo groupadd admin_group
sudo groupadd users_group
```

- Добавление пользователей в группы

```bash
sudo usermod -aG admin_group username
```

---

### 6.5 Настройка прав доступа к ресурсам (1 балл)

- Пример: каталог /srv/data доступен только группе admin_group

```bash
sudo mkdir -p /srv/data
sudo chown root:admin_group /srv/data
sudo chmod 770 /srv/data
```

---

### 6.6 Настройка аутентификации и авторизации (1 балл)

- Настройка PAM (Pluggable Authentication Modules) — пример включения ограничения по времени:

```bash
sudo nano /etc/security/time.conf
```

- Настройка SSH для авторизации по ключам (рекомендуется):

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
nano ~/.ssh/authorized_keys
# Вставьте публичный ключ
chmod 600 ~/.ssh/authorized_keys
```

- В файле `/etc/ssh/sshd_config`:

```
PasswordAuthentication no
PubkeyAuthentication yes
```

- Перезапуск ssh:

```bash
sudo systemctl restart ssh
```

---

### 6.7 Настройка журнала мониторинга (1 балл)

- Используем `rsyslog` (обычно установлен):

```bash
sudo systemctl status rsyslog
```

- Настройка фильтров и отправка логов в отдельный файл, например, для ssh:

```bash
sudo nano /etc/rsyslog.d/50-default.conf
```

Добавить:

```
auth,authpriv.*    /var/log/auth.log
```

- Перезапустить rsyslog

```bash
sudo systemctl restart rsyslog
```

- Просмотр лога:

```bash
tail -f /var/log/auth.log
```

Созданы группы пользователей
1. sudo groupadd название
Эта команда создает новую группу с именем, указанным вместо "название". Например, если вы хотите 
создать группу с именем "developers", команда будет выглядеть так: sudo groupadd developers.

2. sudo adduser testuser
Эта команда создает нового пользователя с именем "testuser". В процессе выполнения команды вам 
может быть предложено задать пароль и некоторые другие параметры (например, полное имя).

3. sudo usermod -aG sudo(группа) testuser
Эта команда добавляет пользователя "testuser" в группу "sudo". Флаг -a (append) означает, что пользователь 
будет добавлен в группу без удаления его из других групп. Это позволит "testuser" выполнять команды с 
правами суперпользователя, используя команду sudo.

4. getent group developers
Если группа существует, вы получите строку с информацией о ней, которая будет выглядеть примерно так:

5. groups имя_пользователя
Проверка групп пользователя

6. sudo gpasswd -d имя_пользователя имя_группы 
Удаление пользователя из группы

Руководство пользователя 6.4.1 
Документ «Руководство пользователя» должен содержать разделы: 
- введение; 
- назначение и условия применения; 
- подготовка к работе; 
- описание операций; 
- аварийные ситуации;
- рекомендации по освоению. 
6.4.2 В разделе «Введение» приводят: 
- область применения средства автоматизации; 
- краткое описание возможностей средства автоматизации; 
- уровень подготовки пользователя; 
- перечень эксплуатационной документации, с которой необходимо ознакомиться пользователю. 
6.4.3 В разделе «Назначение и условия применения» приводят: 
- виды деятельности, функции, для автоматизации которых предназначено данное средство автоматизации; 
- условия применения средства автоматизации в соответствии с назначением (например, характеристики и конфигурация технических средств, операционная среда и общесистемные программные средства, входная информация, требования к подготовке специалистов и т. п.). 
6.4.4 В разделе «Подготовка к работе» приводят: 
- состав и содержание носителя данных, содержащего загружаемые программы и данные; 
- порядок загрузки программ и данных; 
- порядок проверки работоспособности. 
6.4.5 В разделе «Описание операций» приводят: 
- описание всех выполняемых функций, задач (комплексов задач), процедур; 
- описание операций технологического процесса обработки данных, необходимых для выполнения функций, задач (комплексов задач), процедур. 
6.4.6 Для каждой операции приводят: 
- наименование; 
- условия, при соблюдении которых возможно выполнение операции; 
- подготовительные действия; 
- основные действия в требуемой последовательности; 
- заключительные действия; 
- ресурсы, расходуемые на операцию. 
6.4.7 В разделе «Аварийные ситуации» приводят: 
- действия в случае несоблюдения условий выполнения технологического процесса, в том числе при длительных отказах технических средств; 
- действия по восстановлению программ и/или данных при отказе носителей данных или обнаружении ошибок в данных; 
- действия в случаях обнаружения несанкционированного доступа к данным; 
- действия в других аварийных ситуациях. 
6.4.8 В разделе «Рекомендации по освоению» приводят рекомендации по освоению и эксплуатации средства автоматизации, включая описание контрольного примера, правила его запуска и выполнения.
